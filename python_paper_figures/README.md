# Recovery figures

This folder reproduces the diffusivity-recovery figures in the paper. Each experiment
recovers a spatially varying diffusion coefficient `D(x)` and a scalar source strength
`b0` from a single steady-state particle snapshot, using three methods that share a
variable-projection outer loop: discretize-then-optimize (DTO), a physics-informed neural
network (PINN), and bilevel local operator learning (BiLO). The methods are compared over
100 trials per condition.

The pipeline has two steps: run the experiments to produce per-condition `.pkl` result
files, then render the figures from those files. The `.pkl` data and the rendered PDFs are
not tracked in git, since they are large and fully regenerated by the scripts below.

## Requirements

The scripts import the framework from the repository root, so run them from inside this
folder. Python 3.10 or newer, with:

```
pip install torch numpy scipy matplotlib
```

For a CPU-only cluster, install the CPU build of torch, for example
`pip install torch --index-url https://download.pytorch.org/whl/cpu`.

## 1. Generate the result files

Each experiment is one condition, defined by the stochastic convention, the source
strength `B0`, and the `D(x)` spatial frequency `FREQ`. The condition is read from
environment variables, and the script writes
`CCombined_<convention>_b<B0>_t<N>_<FREQ>_data.pkl`. The five manuscript conditions are:

| convention | B0   | FREQ | D(x)                | figure                     |
|------------|------|------|---------------------|----------------------------|
| Ito        | 250  | 4    | 0.1 + 0.05 sin(8πx) | Ito (Fig. 6)               |
| Ito        | 1000 | 4    | 0.1 + 0.05 sin(8πx) | Ito (Fig. 6)               |
| Fickian    | 1000 | 4    | 0.1 + 0.05 sin(8πx) | Fickian high frequency (Fig. 7) |
| Fickian    | 250  | 3    | 0.1 + 0.05 sin(6πx) | Fickian low frequency (Fig. 8)  |
| Fickian    | 1000 | 3    | 0.1 + 0.05 sin(6πx) | Fickian low frequency (Fig. 8)  |

### On a cluster (SLURM)

`run_recovery_sweep.slurm` runs all five conditions as a job array, 100 trials each:

```
sbatch run_recovery_sweep.slurm
```

Set the `#SBATCH` account, partition, and email lines for your site first.
`run_inference.slurm` runs the two default-parameter conditions instead. The one-time
conda environment setup is documented at the top of `run_inference.slurm`.

### Locally

Set the condition with environment variables and run the matching script:

```
N_TRIALS=100 B0=1000 FREQ=4 python Combined_Ito_slurm.py
N_TRIALS=100 B0=250  FREQ=3 python Combined_Fickian_slurm.py
```

`Combined_Ito.py` and `Combined_Fickian.py` are non-cluster variants with the parameters
set directly in the script.

## 2. Render the figures

Once all five `.pkl` files are present, build the three manuscript figures:

```
python plot_recovery_figures.py
```

This writes `fig_ito_recovery.pdf`, `fig_fickian_highfreq.pdf`, and
`fig_fickian_lowfreq.pdf`. Each panel row shows one condition: the recovered `D(x)` (true
profile, per-method median, and 10th to 90th percentile band over the trials) on the left,
and the distribution of the inferred source strength `b0` on the right.
